Feedback to Foresight: Simplifying App Review Sentiment Analysis

This collaborative project focused on employing Natural Language Processing (NLP) for sentiment analysis of user feedback from Google Play Store applications. My primary role as a data engineer involved orchestrating the ETL process and contributing to the optimization of the model. The comprehensive workflow of the project encompassed:

  • Extracting and transforming data from an AWS S3 bucket.
  • Loading the data back into the S3 bucket for the project members to access.
  • Conducting exploratory data analysis to gain insights into the data.
  • Undertaking feature engineering to prepare the data for modeling.
  • Implementing an NLP model to analyze user feedback.
  • Deploying the model on HuggingFace for broader accessibility.

The project resulted in an automated, scalable sentiment analysis model, with potential applications in quality control and early trend detection.

Indonesia's SDG7 Progress

In this project, I worked with Docker to containerize every step of the ETL pipeline and created visualization with Kibana. The project description entails:

  • Developed an ETL process using Apache Airflow to aggregate and clean data pertaining to Indonesia’s progress in sustainable energy (SDG7).
  • Focused on containerization using Docker to encapsulate the ETL process, ensuring a streamlined and reproducible workflow.
  • Cleaned and structured large datasets for efficient analysis, pushing the processed data to Elasticsearch for in-depth analysis.

The result of the project showed that Indonesia’s economic growth has not been fully leveraged to boost investment in renewable energy infrastructure.

Jellyfish Species Classification

In this project, I created a classification model to perform classification on six jellyfish species. The species include:

  • Barrel jellyfish
  • Compass jellyfish
  • Mauve Stinger jellyfish
  • Blue jellyfish
  • Lions mane jellyfish
  • Moon jellyfish

Unfortunately, the best model with around 70% accuracy could not be deployed due to its huge file size (around 1+ GB). So the second best model with around 50% accuracy was deployed instead.
